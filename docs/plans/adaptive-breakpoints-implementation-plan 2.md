**Дата создания:** 2026-03-07

**Статус:** На утверждение

## 1. Текущее состояние адаптивной системы

### 1.1 Адаптивные токены и брейкпоинты
Система включает 9 брейкпоинтов (`sm`, `md`, `lg`, `xl`, `2xl`, `3xl`, `4xl`, `5xl`, `6xl`), определённых в [`adaptive-tokens.ts`](nextjs-project/src/lib/adaptive-tokens.ts). Поддерживаются экраны до 5K+ (5120px). Конфигурация Tailwind расширена соответствующими классами.

### 1.2 Адаптивные компоненты
Существует 4 основных компонента:
1. **`AdaptiveContainer`** – контейнеры с адаптивной шириной и отступами
2. **`ResponsiveText`** – типографика с масштабированием на больших экранах
3. **`FluidGrid`** – адаптивная сетка с поддержкой 9 брейкпоинтов
4. **`ScalableSpacing`** – отступы, увеличивающиеся на больших экранах

### 1.3 Текущий уровень внедрения
**Высокий уровень покрытия:**
- Главная страница (`/`) – полное использование адаптивных компонентов
- Страница "О нас" (`/o-nas`) – полное использование
- Страница "Отзывы" (`/otzyvy`) – полное использование
- Страница "Контакты" (`/contacts`) – полное использование
- Страница "Каталог" (`/catalog`) – полное использование
- Страница "Корзина" (`/cart`) – полное использование
- Страница "FAQ" (`/faq`) – полное использование
- Страница "Новости" (`/news`) – полное использование

**Смешанный подход (частичное внедрение):**
- Страница товара (`/product/[slug]`) – адаптивные компоненты + стандартные классы
- Административные интерфейсы – минимальное использование адаптивных компонентов
- Некоторые статические страницы (Оферта, Политика конфиденциальности и др.) используют стандартные Tailwind классы

**Полное отсутствие адаптивных компонентов:**
- Страницы личного кабинета (`/account/*`)
- Страницы сравнения товаров (`/compare`)
- Wishlist (`/wishlist`)
- Страницы сертификатов и сотрудничества

## 2. Приоритизация страниц и компонентов

### 2.1 Критерии приоритизации
1. **Трафик пользователей** – страницы с наибольшим посещением
2. **Критичность для бизнеса** – конверсионные страницы
3. **Сложность внедрения** – объём изменений
4. **Текущее состояние** – степень готовности

### 2.2 Приоритетные группы

#### Группа 1 (Высший приоритет) – Конверсионные страницы
- **Страница товара** (`/product/[slug]`) – максимальный трафик, ключевая для продаж
- **Страница корзины** (`/cart`) – уже покрыта, требуется полная проверка
- **Страница оформления заказа** (если отдельная) – не обнаружена в структуре
- **Главная страница** (`/`) – уже покрыта, требует улучшения 5xl/6xl поддержки

#### Группа 2 (Высокий приоритет) – Навигационные страницы
- **Личный кабинет** (`/account/*`) – все подстраницы
- **Wishlist** (`/wishlist`) – высокая вовлеченность пользователей
- **Сравнение товаров** (`/compare`) – инструмент принятия решений

#### Группа 3 (Средний приоритет) – Информационные страницы
- **Статьи и новости** (`/news/[slug]`) – уже частично адаптивны
- **Сертификаты соответствия** (`/sertifikaty-sootvetstviya`)
- **Сотрудничество** (`/sotrudnichestvo`)
- **Оферта и Политика конфиденциальности**

#### Группа 4 (Низкий приоритет) – Административные интерфейсы
- **Панель администратора** (`/admin/*`) – внутренний инструмент

#### Группа 5 (Компоненты) – Глобальные компоненты
- **Header** – уже адаптивен (`SiteHeader`, `AdaptiveNav`)
- **Footer** – уже адаптивен (`SiteFooter`)
- **Навигация мобильная/десктопная** – уже адаптивна
- **Карточки товаров** – требуют проверки поддержки 5xl/6xl
- **Формы** – требуют адаптивных отступов и типографики

## 4. Рекомендации по миграции стандартных Tailwind классов

### 4.1 Систематизация замен

| Стандартный класс | Адаптивный компонент | Примечания |
|------------------|----------------------|------------|
| `max-w-[min(90rem,92vw)] mx-auto px-4 sm:px-6 lg:px-8` | `<AdaptiveContainer maxWidth="default">` | Стандартный контейнер |
| `max-w-3xl mx-auto px-4` | `<AdaptiveContainer maxWidth="xl">` | Узкий контейнер |
| `text-2xl font-bold` | `<ResponsiveText variant="2xl" weight="bold">` | С автоматическим масштабированием |
| `text-lg` | `<ResponsiveText variant="lg">` | |
| `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4` | `<FluidGrid cols={2} colsTablet={3} colsDesktop={4}>` | С поддержкой 5xl/6xl |
| `space-y-4` | `<ScalableSpacing size="md" direction="vertical">` | Адаптивные отступы |
| `my-8` | `<ScalableSpacing size="lg" direction="vertical">` | |
| `gap-4` | Встроено в `FluidGrid` через `adaptiveGap` prop | |

### 4.2 Процесс миграции

**Фаза 1: Подготовка (1 день)**
- Создать скрипт для поиска стандартных классов
- Подготовить документацию для команды
- Создать компоненты-заглушки для тестирования

**Фаза 2: Постраничная миграция (7-10 дней)**
- Мигрировать по одной странице из Группы 1
- Проводить тестирование после каждой страницы
- Использовать подход "компонент за компонентом"

**Фаза 3: Проверка и тестирование (3-5 дней)**
- Кросс-браузерное тестирование
- Тестирование на реальных устройствах
- Проверка производительности

### 4.3 Правила для разработчиков

1. **Новые страницы:** Обязательно использовать адаптивные компоненты
2. **Рефакторинг:** При любом изменении существующей страницы мигрировать её на адаптивные компоненты
3. **Исключения:** Административные интерфейсы могут использовать стандартные классы, но с поддержкой 3xl/4xl брейкпоинтов
4. **Тестирование:** Все изменения должны проходить тесты на адаптивность

## 6. Документация для разработчиков

### 6.1 Быстрый старт

**Установка и использование:**
```tsx
import { 
  AdaptiveContainer, 
  ResponsiveText, 
  FluidGrid, 
  ScalableSpacing 
} from '@/components/ui';

// Контейнер с адаптивной шириной
<AdaptiveContainer maxWidth="default">
  <ResponsiveText variant="h1">Заголовок</ResponsiveText>
  <FluidGrid cols={2} colsDesktop={4}>
    {/* Содержимое сетки */}
  </FluidGrid>
  <ScalableSpacing size="lg" />
</AdaptiveContainer>
```

### 6.2 Руководство по миграции

**Пошаговая инструкция:**
1. Определите, какие стандартные классы используются на странице
2. Замените контейнеры на `AdaptiveContainer`
3. Замените текстовые элементы на `ResponsiveText`
4. Замените сетки на `FluidGrid`
5. Добавьте адаптивные отступы через `ScalableSpacing`
6. Протестируйте на всех брейкпоинтах

**Пример миграции:**
```tsx
// ДО
<div className="max-w-3xl mx-auto px-4">
  <h1 className="text-3xl font-bold mb-6">Заголовок</h1>
  <div className="grid grid-cols-2 md:grid-cols-3 gap-4">
    {/* элементы */}
  </div>
</div>

// ПОСЛЕ
<AdaptiveContainer maxWidth="xl">
  <ResponsiveText as="h1" variant="3xl" weight="bold" className="mb-6">
    Заголовок
  </ResponsiveText>
  <FluidGrid cols={2} colsTablet={3} gap={4} adaptiveGap>
    {/* элементы */}
  </FluidGrid>
</AdaptiveContainer>
```

### 6.3 Справочник компонентов

**`AdaptiveContainer`:**
```tsx
<AdaptiveContainer
  maxWidth="default" // 'xl' | '2xl' | '3xl' | '4xl' | '5xl' | '6xl' | 'full'
  center={true}      // центрирование
  adaptivePadding={true} // адаптивные отступы
>
```

**`ResponsiveText`:**
```tsx
<ResponsiveText
  as="h1"            // HTML-тег
  variant="4xl"      // размер: xs-6xl
  weight="bold"      // насыщенность
  adaptive={true}    // масштабирование на больших экранах
>
```

**`FluidGrid`:**
```tsx
<FluidGrid
  cols={2}           // мобильные
  colsTablet={3}     // планшет
  colsDesktop={4}    // десктоп
  colsXl={5}         // xl
  cols2xl={6}        // 2xl
  cols3xl={7}        // 3xl
  cols4xl={8}        // 4xl
  adaptiveGap={true} // адаптивные промежутки
>
```

**`ScalableSpacing`:**
```tsx
<ScalableSpacing
  size="lg"          // xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl, 6xl
  direction="vertical" // vertical, horizontal, top, bottom
/>
```

### 6.4 Частые ошибки и решения

**Проблема:** Текст слишком мелкий на 4K экранах  
**Решение:** Убедитесь, что `adaptive={true}` установлен для `ResponsiveText`

**Проблема:** Контейнеры не центрируются  
**Решение:** Проверьте `center={true}` у `AdaptiveContainer`

**Проблема:** Сетка ломается на определенном брейкпоинте  
**Решение:** Укажите явные значения для всех требуемых брейкпоинтов

## 8. Предложения по улучшению системы адаптивных компонентов

### 8.1 Расширение токенов

**Текущие ограничения:**
- Нет токенов для адаптивных радиусов скругления
- Нет токенов для адаптивных теней
- Ограниченный набор масштабных коэффициентов

**Предлагаемые улучшения:**
1. Добавить токены для `borderRadius`, `boxShadow`, `opacity`
2. Ввести семантические токены (например, `spacing.section` вместо фиксированных значений)
3. Добавить поддержку кастомных CSS-переменных для тем

### 8.2 Новые компоненты

1. **`AdaptiveImage`** – компонент для адаптивных изображений с разными версиями для разных брейкпоинтов
2. **`ResponsiveIcon`** – иконки, масштабирующиеся пропорционально тексту
3. **`FluidForm`** – формы с адаптивными полями и отступами
4. **`ScalableButton`** – кнопки с адаптивными размерами и отступами

### 8.3 Инструменты разработки

1. **Плагин для VS Code** – автодополнение адаптивных компонентов
2. **Storybook** – документация с просмотром на разных брейкпоинтах
3. **Визуальный редактор** – инструмент для настройки адаптивности без кода

### 8.4 Мониторинг и аналитика

1. **Сбор метрик** – какие брейкпоинты используют пользователи
2. **A/B тестирование** – сравнение разных адаптивных стратегий
3. **Обратная связь** – сбор отзывов о читаемости на больших экранах

### 8.5 Оптимизация производительности

1. **Critical CSS** – разделение стилей по брейкпоинтам
2. **Ленивая загрузка** – загрузка компонентов только при необходимости
3. **Кэширование** – кэширование адаптивных версий для разных устройств

## 10. Заключение

Данный план обеспечивает системный подход к внедрению адаптивных брейкпоинтов на всех страницах проекта. Ключевые преимущества:

1. **Единообразие** – все страницы используют одну систему
2. **Масштабируемость** – поддержка экранов до 5K+
3. **Поддержка** – чёткая документация и правила
4. **Тестируемость** – автоматические тесты на всех брейкпоинтах

Реализация плана займет примерно 3.5 недели и потребует участия одного фронтенд-разработчика на полную ставку. Результатом будет полностью адаптивный интерфейс, обеспечивающий оптимальный пользовательский опыт на любом устройстве.
