# Реализация системы управления разделами (категориями) товаров

## Обзор реализованной функциональности

В рамках реализации функциональности управления разделами (категориями) товаров были выполнены следующие задачи:

### 1. Модель данных Prisma

Создана новая модель `Category` с полями:
- `id`: уникальный идентификатор
- `title`: название категории (обязательное)
- `slug`: URL-дружелюбный идентификатор
- `image`: URL изображения категории
- `sortOrder`: порядок сортировки
- `createdAt` и `updatedAt`: даты создания и обновления

Создана промежуточная таблица `ProductCategory` для реализации связи "многие ко многим" между продуктами и категориями.

### 2. CRUD операции для категорий

Реализованы серверные действия для работы с категориями:
- Получение списка всех категорий
- Получение отдельной категории
- Создание новой категории
- Обновление категории
- Удаление категории (с отвязкой от товаров)

### 3. Страница управления категориями

Создана страница `/admin/catalog/categories` с функционалом:
- Отображение списка всех категорий с колонками: Название, Slug, Кол-во товаров
- Возможность сортировки (через поле sortOrder)
- Кнопка "Добавить раздел" для создания новых категорий
- Поля: Название (обязательно), Slug (автогенерация из названия, но редактируемый), Картинка
- Возможность редактирования и удаления категорий

### 4. Мульти-селект категорий в форме товара

Обновлена форма редактирования/создания товара для работы с мульти-селектом категорий:
- Заменен текстовое поле "Категория" на компонент Multi-Select
- Добавлена возможность быстро создать новую категорию прямо из карточки товара
- Выбранные категории отображаются списком с возможностью быстрого удаления

### 5. Фильтрация товаров по категориям

Создан компонент `CategorySidebar` для фильтрации товаров:
- Вертикальный список разделов
- Счетчики товаров рядом с названием раздела
- Активный раздел выделен жирным или цветом
- Светло-серый фон (bg-gray-50)
- Фильтрация товаров по выбранному разделу

### 6. API endpoint'ы

Обновлены API endpoint'ы для работы с категориями:
- PUT запрос для обновления товара с поддержкой категорий
- POST запрос для создания нового товара с категориями

### 7. Seed скрипт

Создан seed-скрипт для автоматического создания базовых категорий:
- Collagen
- Грибная коллекция
- Нутриенты
- Бульоны
- Подарочные наборы
- Подарочные сертификаты
- Акции (опционально)
- Новинки (опционально)

## Технические детали

### Связь между Product и Category

Для реализации связи "многие ко многим" используется промежуточная таблица `ProductCategory`:
```prisma
model ProductCategory {
  productId String
  categoryId String
  product    Product    @relation(fields: [productId], references: [id])
  category   Category   @relation(fields: [categoryId], references: [id])
  @@id([productId, categoryId])
}
```

### API взаимодействие

- При обновлении товара передается массив `categoryIds` вместо одного значения `category`
- При создании товара также передается массив `categoryIds`
- Сервер обновляет связи в промежуточной таблице `ProductCategory`

### UI/UX особенности

- Сайдбар каталога повторяет стиль Tilda с светло-серым фоном
- Активный раздел выделяется жирным шрифтом
- Выбранные категории в форме товара отображаются списком с крестиками для удаления
- Использованы компоненты Shadcn/UI для соответствия дизайну

## Инструкция по запуску

1. Запустить миграцию базы данных:
   ```bash
   npx prisma migrate dev --name add_categories
   ```

2. Запустить seed скрипт для базовых категорий:
   ```bash
   npm run seed:categories
   ```

3. Запустить приложение:
   ```bash
   npm run dev
   ```

## Доступ к функциональности

- Админка: `/admin`
- Управление категориями: `/admin/catalog/categories`
- Каталог товаров: `/admin/catalog`
- Создание товара: `/admin/products/new`
- Редактирование товара: `/admin/products/[id]/edit`
