3.1. Управление разделами (CRUD) — реализовано

В админке: **Каталог** → **Разделы** (страница `/admin/catalog/categories`). Список категорий, создание, редактирование, удаление (товары не удаляются). Ниже — исходное ТЗ.

    Просмотр: Список всех разделов с колонками: Название, Slug, Кол-во товаров. Возможность сортировки (Drag-and-drop или поле sortOrder).

    Создание: Кнопка "Добавить раздел". Поля: Название (обязательно), Slug (автогенерация из названия, но редактируемый), Картинка.

    Редактирование: Изменение названия, slug, картинки.

    Удаление: Удаление раздела (товары при этом НЕ удаляются, просто отвязываются).

Список разделов по умолчанию (Seed):

    Collagen

    Грибная коллекция

    Нутриенты

    Бульоны

    Подарочные наборы

    Подарочные сертификаты

    Акции (опционально)

    Новинки (опционально)

3.2. Привязка товаров к разделам — реализовано

В форме редактирования/создания товара (/admin/products/[id]/edit, /admin/products/new) используется мульти-селект категорий (CategoryMultiSelect). Ниже — исходное ТЗ.

    Заменить текстовое поле "Категория" на компонент Multi-Select (или список с чекбоксами), как на скриншоте image_5f5adc.png.

    Логика:

        Администратор может выбрать один или несколько разделов.

        Должна быть возможность быстро создать новый раздел прямо из карточки товара (кнопка "+ Добавить в раздел").

        Отображение выбранных разделов списком с возможностью быстрого удаления (крестик), как на скрине Tilda (см. блок "РАЗДЕЛЫ КАТАЛОГА" на image_5f5adc.png).

3.3. Фильтрация в списке товаров — реализовано

На странице каталога (/admin/catalog): сайдбар с категориями (CategorySidebar), фильтр по разделу, счётчики товаров. Ниже — исходное ТЗ.

    Добавить Сайдбар (слева) или выпадающий фильтр "Раздел".

    При клике на "Collagen" в таблице должны остаться только товары из этой категории.

    Пункт "Все" сбрасывает фильтр.

    Отображать счетчик товаров рядом с названием раздела в сайдбаре (как на image_5f5ada.jpg: "Collagen 11", "Грибная коллекция 14").

4. UI/UX Требования (Shadcn + Tailwind)

    Сайдбар каталога: Визуально повторить стиль Tilda (см. image_5f5ada.jpg).

        Светло-серый фон (bg-gray-50 или подобный).

        Активный раздел выделен жирным или цветом.

        Список разделов: вертикальный.

    Карточка товара:

        Блок выбора категорий должен быть понятным. Используй компонент Command (Combobox) из Shadcn/ui для поиска и выбора категорий.

        Выбранные категории выводятся списком под инпутом (см. скриншот).

5. План реализации (для AI)

    Backend:

        Обновить schema.prisma.

        Выполнить миграцию (npx prisma migrate dev --name add_categories).

        Написать Seed-скрипт для создания базовых категорий.

        Обновить Server Actions (createProduct, updateProduct) для приема массива categoryIds.

    Frontend (Admin):

        Создать страницу управления категориями /admin/catalog/categories.

        Создать компонент CategorySidebar для страницы списка товаров.

        Обновить форму товара (ProductForm), добавив мульти-селект категорий.
