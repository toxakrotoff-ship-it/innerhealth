# Состав заказа в заявках с Тильды

Страница заявок с Тильды в админке: **/admin/tilda-leads**. Импорт: скрипт `npm run import-tilda-leads` (nextjs-project).

В экспорте лидов Тильды (CSV) **нет отдельной колонки** с составом заказа и суммой. В интерфейсе Тильды в карточке лида заказ отображается полностью, но в выгрузке эти данные могут попадать только в произвольные поля формы.

## Где может быть состав заказа

### 1. Поля Input / Input_2 в экспорте

Часто форма корзины Тильды передаёт состав заказа в одно из полей формы (например, скрытое поле с JSON). Тогда в CSV это будет колонка **Input** или **Input_2**.

- Эти поля импортируются в `TildaLead.input` и `TildaLead.input2`.
- В админке («Заявки с Тильды» → «Подробнее») мы пытаемся распознать JSON: если там массив объектов с названием товара, ценой и количеством — показывается блок **«Состав заказа»** с таблицей и итогом.
- Поддерживаются варианты полей: `title`/`name`/`productName`, `quantity`/`qty`/`amount`, `price`/`sum`.

**Что проверить в Тильде:** в настройках формы корзины посмотреть, есть ли поле, в которое подставляется содержимое корзины (например, JSON). Если да — убедиться, что оно попадает в экспорт лидов (в настройках экспорта/полей лида можно добавить нужные колонки).

### 2. Личный кабинет Тильды

В разделе **Лиды** при открытии карточки лида с заказом отображаются контакты, состав заказа, доставка и оплата. Оттуда можно копировать данные вручную. Для массового переноса состава в нашу БД этого недостаточно.

### 3. Отдельный экспорт заказов / платёжной системы

- Если подключена **платёжная система** (ЮKassa, Robokassa и т.д.), в её кабинете обычно есть выгрузка транзакций/заказов с составом. Сопоставление с нашими заявками — по email, дате или внешнему ID (например, `tranid` из Тильды, если он передаётся в платёжку).
- В Тильде в разделе **Магазин** может быть экспорт/отчёт по заказам — стоит проверить наличие выгрузки с товарами и суммами.

### 4. API Тильды

В [документации API Тильды](https://help.tilda.cc/api) можно уточнить, есть ли метод получения полных данных лида (включая заказ) по ID. Если да — можно периодически подтягивать состав заказа по `tildaTranId` или `externalid` и сохранять у нас (например, в том же JSON в полях `input`/`input2` или в отдельной структуре).

### 5. Настройка формы корзины

Чтобы состав и сумма всегда попадали в экспорт лидов:

- Добавить в форму корзины **скрытое поле**, в которое через JavaScript подставляется JSON корзины (массив объектов с полями вроде `title`, `price`, `quantity`).
- Убедиться, что это поле отображается в списке полей лида и включается в экспорт (настройки полей в разделе «Лиды»).

После следующего экспорта CSV и повторного импорта скриптом `import-tilda-leads.ts` в админке в «Подробнее» по таким заявкам автоматически появится блок «Состав заказа» с таблицей и итогом.
