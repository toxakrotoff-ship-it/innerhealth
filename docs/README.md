## Обзор проекта

InnerHealth — интернет-магазин спортивного питания и добавок, построенный на современном стеке технологий.

### 🛠 Стек технологий

| Компонент | Технология |
|-----------|------------|
| **Фронтенд** | Next.js 16 (App Router), React, TypeScript |
| **Стилизация** | Tailwind CSS + Shadcn/ui компоненты |
| **Бэкенд** | Next.js API Routes, Server Actions |
| **База данных** | PostgreSQL + Prisma ORM |
| **Аутентификация** | NextAuth v4 (Credentials, JWT) + 2FA (TOTP/email); v5 в планах |
| **Интеграции** | ЮKassa (оплата), СДЭК (доставка), Telegram-бот |
| **Деплой** | Docker, Nginx, Let's Encrypt SSL |
| **Мониторинг** | Логи через Docker, адаптивная аналитика |

### 📁 Структура репозитория

```
innerhealth/
├── nextjs-project/          # Основное приложение Next.js (Prisma, API, UI)
│   ├── src/app/             # App Router
│   ├── src/lib/             # Утилиты, Prisma singleton, auth, интеграции
│   ├── src/services/        # Сервисный слой (DAL)
│   ├── prisma/              # schema.prisma и миграции (источник истины)
│   └── docs/                # Env (SMTP, 2FA, Яндекс.Карты), деплой приложения
├── docs/                    # Проектная документация (эта папка) и docs/plans/
├── tests/                   # Playwright (корень репозитория)
├── prisma/                  # Дублирующая/legacy-схема (при расхождении ориентируйтесь на nextjs-project/prisma)
├── README.md
└── deploy/                  # Nginx, docker-compose и скрипты деплоя
```

## Архитектура

### Монолитная архитектура с разделением ответственности

Проект использует монолитную архитектуру на Next.js с чётким разделением слоёв:

```
┌─────────────────────────────────────────────────┐
│                 Публичная часть                  │
│   (site) группа: /, /catalog, /product, /cart   │
├─────────────────────────────────────────────────┤
│              Админ-панель (protected)            │
│   /admin/* с защитой через middleware + Auth    │
├─────────────────────────────────────────────────┤
│              API Layer (Next.js API Routes)      │
│   /api/* для интеграций (ЮKassa, СДЭК, Telegram) │
├─────────────────────────────────────────────────┤
│             Бизнес-логика (Services)             │
│   account.service.ts, order.service.ts и др.    │
├─────────────────────────────────────────────────┤
│              Data Access Layer (Prisma)          │
│   Модели Product, User, Order, Category и др.   │
└─────────────────────────────────────────────────┘
```

### Ключевые архитектурные решения

1. **Синглтон Prisma** — `src/lib/prisma.ts` для безопасного подключения к БД
2. **Edge / proxy** — `src/proxy.ts` (Next.js 16): авторизация админки/API, CSP, rewrite `ADMIN_SECRET_PATH`, редиректы из БД
3. **Сервисный слой** — бизнес-логика вынесена в отдельные сервисы (`src/services/`)
4. **Адаптивный UI** — компоненты с адаптивными токенами (`src/lib/adaptive-tokens.ts`)
5. **Интеграции через REST API** — модульная структура для ЮKassa, СДЭК, Telegram

Подробнее об архитектуре админки в [adm_now.md](./adm_now.md).

## Функциональные разделы

### Админ-панель

**Основные модули:**
- **Каталог товаров** — таблица с фильтрацией, инлайн-редактирование цены и остатка
- **Товары** — создание/редактирование с табами, SEO-блоком, галереей изображений (D&D)
- **Категории** — древовидная структура, CRUD операции
- **Новости/Статьи** — редактор TipTap, загрузка медиа
- **Промокоды** — создание, редактирование, активация/деактивация
- **Заказы (CRM)** — просмотр состава, статусы, создание отгрузок в СДЭК
- **Пользователи** — управление, приглашение по email, удаление
- **Партнёры** — назначение роли PARTNER, привязка промокодов, комиссия
- **Настройки** — сайт, почта, ЮKassa, СДЭК, Telegram
- **Модерация** — отзывы (одобрение/отклонение), заявки с Тильды, партнёрство

**Технические особенности:**
- Защищённый путь: `/admin` (или настраиваемый через `ADMIN_SECRET_PATH`)
- Session-based авторизация через NextAuth
- Серверные действия (Server Actions) для форм
- Редактор TipTap для контента товаров и новостей

Подробная структура админки в [adm_now.md](./adm_now.md).

### Публичная часть

**Маршруты (группа `(site)`):**
- `/` — главная страница с hero-блоком, категориями, новостями
- `/catalog` — каталог товаров с фильтрацией по категориям
- `/product/[slug]` — карточка товара с галереей, табами, отзывами
- `/cart` — корзина с расчётом доставки СДЭК и применением промокодов
- `/news` — лента новостей и статей
- `/otzyvy` — опубликованные отзывы
- Страницы: "О нас", "Контакты", "Сотрудничество", "Политика", "Оферта"

**Особенности:**
- Адаптивный дизайн с mobile-first подходом
- Корзина в виде drawer с сохранением состояния
- Интеграция Яндекс.Карт для выбора ПВЗ СДЭК
- Оптимизация производительности (lazy loading, image optimization)

### Личный кабинет пользователя

**Доступен по маршруту `/account` для авторизованных пользователей:**

- **Профиль** — редактирование данных, настройка 2FA
- **Заказы** — история заказов, статусы, детализация
- **Адреса доставки** — управление адресами для быстрого оформления
- **Верификация email** — подтверждение почты через код

**Техническая реализация:**
- Защита через `require-user-session` middleware
- Валидация через Zod схемы (`src/lib/validations/`)
- Интеграция с сервисами (`account.service.ts`, `user.service.ts`)

### Личный кабинет партнёра

**Доступен для пользователей с ролью `PARTNER` по маршруту `/account/partner`:**

- **Промокоды** — управление промокодами с настройкой комиссии
- **Статистика** — заказы по промокодам, доход, конверсия
- **Профиль партнёра** — контактные данные, настройки уведомлений

**Особенности:**
- Отдельная бизнес-логика в `partner.service.ts`
- Привязка промокодов через `PartnerPromoCode` с `commissionPercent`
- Доступ из админки для управления партнёрами

### Интеграции

#### ЮKassa (оплата)
- **Реализация:** Создание платежа при оформлении заказа, вебхук для обновления статусов
- **Чеки 54-ФЗ:** Автоматическая генерация чеков с НДС
- **Настройка:** Учётные данные через админку или переменные окружения
- **Документация:** [step_3_yokassa|cdek.md](./step_3_yokassa%7Ccdek.md)

#### СДЭК (доставка)
- **Функции:** Расчёт стоимости, выбор ПВЗ, создание заказов на отгрузку
- **API методы:** Auth, города, ПВЗ, калькулятор, создание заказа
- **Габариты товаров:** Вес и размеры для точного расчёта
- **Документация:** [step_3_yokassa|cdek.md](./step_3_yokassa%7Ccdek.md)

#### Telegram-бот
- **Назначение:** Уведомления администраторов о заказах, модерация отзывов
- **Реализация:** Отдельный сервис в Docker (`npm run telegram-bot`)
- **Функции:** Команды `/start`, `/promo`, кнопки модерации
- **Документация:** [tg_bot.md](./tg_bot.md)

#### Яндекс.Карты
- **Использование:** Карта ПВЗ СДЭК в корзине, карта на странице контактов
- **Ключ API:** `NEXT_PUBLIC_YANDEX_MAPS_API_KEY`
- **Документация:** [nextjs-project/docs/yandex-maps-env.md](../nextjs-project/docs/yandex-maps-env.md)

### Безопасность и аутентификация

#### Аутентификация
- **NextAuth v4** с Credentials провайдером
- **JWT сессии** с кастомными callbacks
- **2FA (двухфакторная аутентификация):**
  - TOTP (Google Authenticator / Authy)
  - Email-коды
  - Настройка в профиле пользователя

#### Защита данных
- **Валидация:** Zod схемы для всех критичных входных данных
- **Rate limiting:** Защита от перебора (в памяти или Redis)
- **Безопасные заголовки:** CSP, HSTS, X-Frame-Options
- **Защита от SSRF:** Проверка URL при загрузке изображений
- **Хеширование:** Пароли через bcrypt, токены срока действия

Полный аудит безопасности в [step_6_security.md](./step_6_security.md).

## Деплой и инфраструктура

### Быстрый старт (разработка)

```bash
cd nextjs-project
cp .env.example .env.local  # заполнить переменные
npm install
npx prisma migrate dev
npm run dev
```

### Деплой на VPS (продакшен)

**Требования:** Ubuntu, Docker, Docker Compose

**Основные шаги:**
1. Клонирование репозитория на сервер
2. Настройка `.env` с продовыми значениями
3. Запуск через Docker Compose

**Два сценария деплоя:**
- **Полный деплой:** `./deploy/deploy.sh` (пересборка всех образов)
- **Быстрое обновление:** `./deploy/deploy-quick.sh` (только код и миграции)

**Конфигурация Docker:**
- `app` — Next.js приложение
- `db` — PostgreSQL
- `nginx` — reverse proxy с SSL
- `telegram-bot` — отдельный сервис бота

Подробное руководство в [deploy-vps.md](./deploy-vps.md).

### Миграции Prisma

- **Автоматически:** При старте приложения (если `RUN_MIGRATE=1`)
- **Вручную:** `docker compose run --rm app npx prisma migrate deploy`
- **Откат:** Через `npx prisma migrate resolve`

Документация по миграциям: [prisma-migrations-and-deploy.md](./prisma-migrations-and-deploy.md).

## Переменные окружения

### Основные переменные

| Переменная | Назначение | Пример |
|------------|------------|--------|
| `DATABASE_URL` | URL подключения к PostgreSQL | `postgresql://user:pass@localhost:5432/db` |
| `NEXTAUTH_URL` | Базовый URL для NextAuth | `https://innerhealth.ru` |
| `NEXTAUTH_SECRET` | Секрет для подписи JWT | 32-символьная случайная строка |
| `ADMIN_SECRET_PATH` | Скрытая часть пути админки | `hidden-admin-url-part` |
| `YOOKASSA_SHOP_ID` | Идентификатор магазина ЮKassa | `your_shop_id` |
| `YOOKASSA_SECRET_KEY` | Секретный ключ ЮKassa | `your_secret_key` |
| `CDEK_CLIENT_ID` | Client ID для API СДЭК | `your_id` |
| `CDEK_CLIENT_SECRET` | Client Secret для API СДЭК | `your_password` |
| `TELEGRAM_BOT_TOKEN` | Токен бота от BotFather | `your_bot_token` |
| `SMTP_HOST` / `USER` / `PASS` | Настройки почтового сервера | `smtp.gmail.com` |
| `NEXT_PUBLIC_YANDEX_MAPS_API_KEY` | Ключ API Яндекс.Карт | `your_yandex_maps_apikey` |

### Группы переменных

1. **База и Auth** — `DATABASE_URL`, `NEXTAUTH_*`
2. **Админка** — `ADMIN_SECRET_PATH`, `ADMIN_2FA_SECRET`
3. **Платежи** — `YOOKASSA_*`
4. **Доставка** — `CDEK_*`, `NEXT_PUBLIC_YANDEX_MAPS_API_KEY`
5. **Уведомления** — `TELEGRAM_*`, `SMTP_*`

Полный список с описанием в [env's.md](./env's.md).
