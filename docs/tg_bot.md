# Telegram-бот для уведомлений Inner Health

## Обзор

Бот привязан к сайту и предназначен для:
1. **Уведомления о новых заказах** (с полной детализацией корзины) и о заявках с любых форм сайта — только пользователям из вайтлиста.
2. **Уведомления о новом отзыве** с кнопками «Разместить» и «Отклонить» (форма отзывов и модель — позже).
3. **Просмотр статистики по промокодам** (сколько раз использовали) по команде в боте.

---

## Подход: Вариант A (отдельный Node-сервис)

- Бот работает как **отдельный процесс** (например, `services/telegram-bot` или `packages/telegram-bot`).
- Приём апдейтов от Telegram — через **long polling** (не нужен публичный HTTPS для входящих запросов от Telegram).
- Сайт (Next.js) только **отправляет** уведомления в Telegram через Bot API (вызов `sendMessage` и т.д.) и не принимает webhook.
- Плюсы: долгие соединения и нагрузка от бота не затрагивают Next.js; проще масштабировать и добавлять команды.

---

## Конфигурация и токен

- Токен бота хранится **только в переменных окружения**, никогда не коммитится в репозиторий.
- Переменная: `TELEGRAM_BOT_TOKEN`.
- Получить токен: [@BotFather](https://t.me/BotFather) → /newbot или /token.
- Для отдельного сервиса бота: тот же `TELEGRAM_BOT_TOKEN` передаётся в процесс бота (env при запуске или общий `.env` в корне монорепозитория).
- **TELEGRAM_SITE_URL** — URL сайта, по которому бот вызывает API (confirm, whitelist, promo-stats). Если бот и сайт в **разных Docker-контейнерах** на одной машине, задайте `http://app:3000` (имя сервиса приложения из docker-compose). В `nextjs-project/docker-compose.yml` для сервиса `telegram-bot` это уже прописано в `environment`.

## Вайтлист: админка и профиль

### Сценарий

1. В **админке** во вкладке **«Профиль»** админ видит блок «Уведомления Telegram».
2. Админ нажимает «Подключить Telegram» (или вставляет ссылку на свой TG-профиль — опционально, см. ниже).
3. Бэкенд генерирует **одноразовый код привязки** (короткий случайный код, привязанный к текущему пользователю и сессии).
4. Админу показывается ссылка на бота с параметром `start`:  
   `https://t.me/<bot_username>?start=<code>`  
   и инструкция: «Нажмите ссылку и отправьте боту команду Start».
5. Пользователь переходит по ссылке, в боте автоматически отправляется `/start <code>`.
6. Бот получает апдейт, извлекает `code` из `message.text`, вызывает **внутренний API сайта** (например, `POST /api/admin/telegram/confirm` с телом `{ code, telegramUserId }`).
7. API проверяет код (существует, не истёк, привязан к админу), затем добавляет или обновляет запись в таблице вайтлиста (например, `TelegramWhitelist`: `userId`, `telegramUserId`, `linkedAt`). Код после успешного использования помечается использованным или удаляется.
8. Бот отвечает пользователю: «Вы добавлены в список уведомлений».

Ссылка на TG-профиль в профиле админа — опционально (для отображения «ваш Telegram» или подсказки); для привязки достаточно одноразового кода и нажатия Start в боте.

### Хранение в БД (Prisma)

- Таблица, например: `TelegramWhitelist`:
  - `id` (cuid)
  - `userId` (FK на `User`) — какой админ привязал
  - `telegramUserId` (BigInt или String) — Telegram user ID (числовой)
  - `linkedAt` (DateTime)
  - Уникальность: один `telegramUserId` — одна запись (или один `userId` — одна запись, если один админ = один Telegram).
- Для генерации кодов привязки — таблица `TelegramLinkCode` (или временное хранилище):
  - `code` (уникальный, короткий)
  - `userId` (FK на `User`)
  - `expiresAt` (DateTime, например 10 минут)
  - после успешного подтверждения запись удаляется или помечается использованной.

### Безопасность вайтлиста

- **Коды привязки**: генерировать криптостойко (crypto.randomBytes); срок жизни 5–10 минут; одноразовые.
- **API подтверждения** (`/api/admin/telegram/confirm`): доступен только авторизованным пользователям с ролью ADMIN (или WRITER, по политике); проверять, что `code` принадлежит текущему `session.user.id`.
- **Бот при подтверждении**: передаёт только `code` и свой `telegramUserId`; сервер сам находит по коду `userId` и проверяет сессию, если вызов идёт из бота с сервисным ключом (см. ниже).
- Альтернатива: бот не вызывает публичный API, а сам имеет доступ к БД (в том же сервере) и проверяет код по таблице `TelegramLinkCode`; тогда код должен быть достаточно длинным и одноразовым.
- В вайтлист добавлять только после успешной проверки кода и прав пользователя.
- Не хранить в логах и не отдавать в API полные коды привязки; в UI показывать только ссылку с кодом.

## Отправка уведомлений с сайта (Next.js)

- Общий модуль, например `lib/telegram-notify.ts`:
  - читает список `telegramUserId` из таблицы вайтлиста (Prisma);
  - форматирует сообщения (заказ, форма, позже — отзыв);
  - вызывает Telegram Bot API: `POST https://api.telegram.org/bot<TOKEN>/sendMessage` (токен из `process.env.TELEGRAM_BOT_TOKEN`).
- Вызовы **не блокируют** ответ пользователю: после успешного сохранения заказа/формы вызывать отправку в фоне (например, `void notifyTelegramOrder(...)` или через микрозадачу), без await в критичном пути ответа.
- Ошибки отправки логировать, не падать с 500 для клиента.

### Заказы

- Точка вставки: в `POST /api/orders` сразу после успешного `prisma.$transaction` (создание заказа и доставки).
- В сообщение включать: номер/ID заказа, состав корзины (название товара, кол-во, цена, сумма по позиции), итог, доставка (ФИО, телефон, адрес, город и т.д.), применённый промокод при наличии.

### Формы

- Точка вставки: после успешного сохранения в БД в каждом обработчике форм (сейчас — `POST /api/partnership`).
- Сообщение: тип формы (например, «Партнёрская заявка»), все поля (имя, email, телефон, роль, сообщение и т.д.).
- При появлении новых форм — аналогично: после сохранения вызывать общий метод с именем формы и данными.

### Отзывы (позже)

- При создании отзыва вызывать `notifyTelegramNewReview(reviewId, authorName, text)`.
- Сообщение в Telegram с двумя inline-кнопками: «Разместить» и «Отклонить» (callback_data: например `review_approve_<id>` и `review_reject_<id>`).
- Обработка нажатий — в процессе бота (см. ниже).

## Обработка в боте (отдельный сервис)

- Long polling: цикл `getUpdates` с `offset`, обрабатывать `message` (команда /start с кодом) и `callback_query` (кнопки по отзывам).
- **Команда /start с кодом**: извлечь код, проверить через API или БД, добавить `telegramUserId` в вайтлист, ответить пользователю.
- **Callback «Разместить»/«Отклонить»**: проверить, что `callback_query.from.id` в вайтлисте; вызвать внутренний API сайта (например, `PATCH /api/admin/reviews/[id]` с `{ status: "approved" | "rejected" }`) с сервисной авторизацией; ответить в Telegram (answerCallbackQuery, при необходимости отредактировать сообщение).
- **Команда статистики промокодов** (например, `/promo`): доступна только если `from.id` в вайтлисте; запрос к API сайта (например, `GET /api/admin/promo-codes`) или к БД; ответ — список промокодов с полем «использований» (usedCount).

### Безопасность при обращении бота к сайту/БД

- Если бот вызывает API сайта: отдельный секрет (например, `TELEGRAM_SERVICE_SECRET` или `INTERNAL_API_KEY`), передавать в заголовке (например, `X-Service-Key`). API маршруты для бота проверяют этот ключ и не полагаются на cookie сессии.
- Токен бота и сервисный ключ — только в env процесса бота и (для отправки) в env Next.js; не в коде и не в репозитории.
- В callback и командах проверять, что `telegramUserId` в вайтлисте, чтобы посторонние не могли вызывать админские действия через бота.

---

## Статистика по промокодам

- В БД уже есть `PromoCode.usedCount` (инкремент при заказе в `/api/orders`).
- Бот по команде (например, `/promo`) запрашивает данные (API или Prisma) и выводит список: код, использований (usedCount), лимит (usageLimit), активен/неактивен.
- Доступ к команде — только для telegram user id из вайтлиста.

## Как добавить себя в вайтлист

1. Зайти в **админку** → **Профиль** (или через меню профиля в шапке → «Редактировать профиль»).
2. В блоке «Уведомления Telegram» нажать **«Подключить Telegram»**.
3. Откроется ссылка на бота с одноразовым кодом (действует ~10 минут). Нажать **«Открыть бота в Telegram»** или скопировать ссылку.
4. В Telegram нажать **Start** (или отправить команду /start). Бот ответит: «Вы добавлены в список уведомлений».
5. При необходимости обновить страницу профиля в админке — статус сменится на «Telegram подключён».

Для работы бота должен быть запущен процесс: `npm run telegram-bot` (из каталога `nextjs-project`). В `.env` должны быть заданы `TELEGRAM_BOT_TOKEN`, `TELEGRAM_SERVICE_SECRET`, `TELEGRAM_SITE_URL` (см. docs/env's.md).
