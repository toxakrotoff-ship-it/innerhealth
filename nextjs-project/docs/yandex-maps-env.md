# Карта пунктов выдачи (Яндекс.Карты)

На странице доставки отображается карта пунктов выдачи СДЭК. Для работы карты нужен API-ключ Яндекс.Карт.

## Как получить ключ

1. Зайдите в [Консоль разработчика Яндекса](https://developer.tech.yandex.ru/).
2. Создайте или выберите приложение.
3. Подключите **JavaScript API и HTTP Геокодер** (достаточно для карты).
4. Создайте ключ (**Новый ключ**), скопируйте его и при необходимости ограничьте по доменам (см. ниже).

### Ограничения ключа (обязательно для JavaScript API 3.0)

Если вы используете **JavaScript API версии 3.0**, в консоли Яндекса нужно настроить ограничения для ключа — иначе API может не работать. В более старых версиях ограничения не обязательны, но их рекомендуют для защиты ключа.

В разделе «Ключи API» нажмите иконку настроек (глаз/ограничения) у нужного ключа и укажите, например:

- **HTTP-рефереры (источники)** — домены, с которых разрешена загрузка карты. Указывать **только доменное имя**, без протокола и пути, по одному в строку, например:
  - `ваш-сайт.ru`
  - `www.ваш-сайт.ru`
  - `localhost` — для локальной разработки

Если домен к сайту ещё не подключён (сайт открывается по IP), можно временно оставить список рефереров пустым — ключ будет работать с любого источника. После подключения домена добавьте его в настройках ключа.

Подробнее: [ограничения и их настройка](https://yandex.ru/dev/maps/jsapi/doc/3.0/concepts/security.html) в документации Яндекс.Карт.

## Где задать переменную

### Локальная разработка

В корне `nextjs-project` создайте или дополните `.env.local`:

```env
NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш_ключ_здесь
```

После изменения перезапустите `npm run dev`. Переменные с префиксом `NEXT_PUBLIC_` подставляются в клиентский код на этапе сборки.

### Сборка и запуск через Docker

Ключ должен быть доступен **на этапе сборки** образа (Next.js вшивает `NEXT_PUBLIC_*` в бандл).

**Вариант 1 — через `.env` в корне проекта:**

```env
NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш_ключ_здесь
```

При запуске `docker compose build` и `docker compose up` переменная подхватится (в `docker-compose.yml` она передаётся как build-arg).

**Вариант 2 — передать при сборке вручную:**

```bash
export NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш_ключ
docker compose build --build-arg NEXT_PUBLIC_YANDEX_MAPS_API_KEY=$NEXT_PUBLIC_YANDEX_MAPS_API_KEY
docker compose up -d
```

## Как проверить наличие ключа на VPS

Ключ подставляется при **сборке** образа (Next.js вшивает `NEXT_PUBLIC_*` в клиентский бандл). Проверки:

1. **Файл `.env` на сервере** (в каталоге, откуда запускаете `docker compose`):
   ```bash
   grep NEXT_PUBLIC_YANDEX_MAPS_API_KEY .env
   ```
   Должна вывестись строка с ключом (значение может быть пустым — тогда ключ не задан).

2. **Что передаётся в образ при сборке:**
   ```bash
   cd nextjs-project
   docker compose config | grep -A2 YANDEX
   ```
   Должна быть переменная `NEXT_PUBLIC_YANDEX_MAPS_API_KEY` в `build.args` у сервиса `app`.

3. **Есть ли переменная в уже собранном образе** (если в Dockerfile она проброшена как `ENV`):
   ```bash
   docker compose run --rm app printenv NEXT_PUBLIC_YANDEX_MAPS_API_KEY
   ```
   Пустой вывод значит, что при сборке ключ не передавали или не сохранили в образ.

4. **Практическая проверка:** откройте сайт, откройте DevTools → вкладка Network, обновите страницу с картой. Найдите запрос к `api-maps.yandex.ru` — в URL будет параметр `apikey=...`. Если такого запроса нет или он с ошибкой, ключ не подставился или неверный.

Если ключ добавили или изменили в `.env`, нужна **пересборка** образа и перезапуск: `docker compose build --no-cache app && docker compose up -d`.

## Если карта не загружается

### Ключ в .env есть, ограничений не было — почему не работает?

`NEXT_PUBLIC_*` подставляется в клиентский код **только при сборке** (`npm run build`). В Docker образ ключ попадает лишь если при `docker compose build` переменная была задана (из `.env` в каталоге сборки). Если образ собирали до добавления ключа или без него — в бандле окажется пустая строка, и карта не загрузится.

**Решение:** пересобрать образ и перезапустить контейнеры **из того же каталога, где лежит `.env` с ключом** (например `cd /opt/innerhealth/nextjs-project`):

```bash
# Убедитесь, что в текущем каталоге есть .env с ключом
grep NEXT_PUBLIC_YANDEX_MAPS_API_KEY .env

# Сборка подхватывает переменные из .env в текущем каталоге
docker compose build --no-cache app
docker compose up -d
```

Если собираете из другого каталога (например родительского), переменная может не подставиться — тогда передайте явно: `NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш_ключ docker compose build --no-cache app`.

После пересборки в браузере во вкладке Network должен появиться запрос к `api-maps.yandex.ru` с параметром `apikey=ваш_ключ`. Если запроса нет — ключ в бандл не попал, проверьте путь и наличие переменной при сборке.

### Скрипт блокируется CSP (в Network — тип «CSP», 0 байт)

В проекте включён Content-Security-Policy (edge-файл Next.js 16: `src/proxy.ts` — см. [middleware-to-proxy](https://nextjs.org/docs/messages/middleware-to-proxy)). Для карты в CSP разрешены: `script-src` — `https://api-maps.yandex.ru` и `https://yastatic.net` (Яндекс подгружает бандл с yastatic.net; без него в консоли: «заблокировали выполнение сценария на …yastatic.net…» и «ymaps: Failed to bundle "full"»); `connect-src` — `https://api-maps.yandex.ru` и `https://*.maps.yandex.ru`. После изменения пересоберите образ и перезапустите приложение.

### Другие причины

- **«Проверьте NEXT_PUBLIC_YANDEX_MAPS_API_KEY»** — скрипт Яндекс.Карт не загрузился. Возможные причины:
  - ключ не попал в сборку (см. выше);
  - блокировка CSP (см. выше);
  - ключ неверный или отозван;
  - в консоли разработчика Яндекса не подключён JavaScript API;
  - блокировка запросов к `api-maps.yandex.ru` (сеть, расширения браузера).

Убедитесь, что ключ указан в нужном месте и после изменения окружения выполнена пересборка/перезапуск приложения.
