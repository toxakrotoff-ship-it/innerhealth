# 2FA (двухфакторная аутентификация)

Включение и настройка 2FA — в админке: **Настройки** → блок «Двухфакторная аутентификация». Доступны способы: код по email или TOTP (Google Authenticator и аналоги).

## Переменные окружения

### TOTP (приложение)

Для способа «через приложение» секрет TOTP хранится в БД в зашифрованном виде. Нужен ключ шифрования (32 байта, base64):

```env
# Ключ шифрования секрета TOTP (32 байта, base64)
TOTP_SECRET_ENCRYPTION_KEY=...
```

Сгенерировать ключ:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

Без `TOTP_SECRET_ENCRYPTION_KEY` включить 2FA через приложение нельзя (ошибка при сохранении секрета).

### Email (код по почте)

Для способа «по email» коды отправляются через тот же SMTP, что и сброс пароля и уведомления о заказах. Используются переменные:

- `SMTP_HOST`, `SMTP_PORT`, `SMTP_SECURE`, `SMTP_USER`, `SMTP_PASS`
- `SUPPORT_EMAIL_FROM` (или по умолчанию support@innerhealth.ru) — от кого письмо с кодом

Без настроенного SMTP коды по email не отправляются; пользователю будет ошибка при входе с 2FA по email.

### Cookie и сессия

Подпись cookie «ожидание 2FA» и выдача одноразового grant используют `NEXTAUTH_SECRET`. Он должен быть задан (как и для обычного входа).

## Поведение

- **Без 2FA:** вход как раньше (email + пароль).
- **С 2FA:** после ввода пароля редирект на страницу ввода кода; код приходит на email или вводится из приложения; после верного кода создаётся сессия.
- **Отключение 2FA:** в Настройках — «Отключить 2FA»; для подтверждения нужен пароль или текущий код из приложения.
